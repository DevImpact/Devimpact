## ========================

## 1️⃣ متطلبات اللعبة الأساسية

## ========================

* اللعبة **ثلاثية الأبعاد (3D)**.
* التطوير يتم عبر **Android Studio**.
* لغة البرمجة الأساسية:

  * **Kotlin (أساسي)** أو **Java (عند الضرورة)**.
* تقنيات العرض:

  * OpenGL ES / Vulkan / أو أي Rendering Layer متوافق مع Android.
* Package Name:

```
com.game.اسم_اللعبة.devimpact
```

* تحتوي اللعبة على **عدد كبير من المراحل (Levels)**.
* كل مرحلة تختلف عن الأخرى في:

  * التصميم البصري
  * الميكانيكيات
  * منطق اللعب
  * مستوى الصعوبة
* الصعوبة **متدرجة بشكل صارم**:

```
Easy → Medium → Hard → Expert
```

* وجود نظام تقدم واضح:

  * Unlocking Levels
  * Progress Persistence
* وجود نظام:

  * Win Conditions
  * Lose Conditions
* واجهة مستخدم HUD احترافية:

  * نقاط
  * وقت
  * محاولات
  * صحة / طاقة (إن وُجد)
* دعم:

  * Restart Level
  * Pause / Resume
* الكود يجب أن يكون:

  * نظيف
  * مقسم
  * قابل للتوسع
  * قابل للاختبار

## ========================

## 2️⃣ قواعد صارمة في البرمجة

## ========================

❗ قواعد غير قابلة للنقاش ❗

* ❌ ممنوع تكرار الكود (NO Code Duplication).
* الالتزام الصارم بـ **SOLID Principles**.
* اعتماد **Clean Architecture**:

```
Presentation
Domain
Data
```

* فصل صارم بين:

```
Game Logic ≠ Rendering ≠ UI
```

* أي ميكانيك يجب أن يكون:

  * Modular
  * Reusable
  * Decoupled
* جميع القيم القابلة للتعديل:

  * تُدار عبر Config Classes / Data Models
  * ❌ ممنوع Hard Coding للأرقام
* لا منطق لعب داخل:

  * Activities
  * Fragments
  * Views

## ========================

## 3️⃣ نظام المراحل (Levels System)

## ========================

* كل مرحلة لها:

  * تعريف بيانات مستقل (LevelConfig)
  * إعدادات صعوبة خاصة
  * تحدٍ أو ميكانيك فريد
* استخدام **Level Manager مركزي** مسؤول عن:

  * تحميل المراحل
  * الانتقال بينها
  * التحقق من التقدم
* دعم:

  * Checkpoints (للمراحل المتقدمة)
  * Resume Level
* يجب أن يكون النظام:

  * Data-Driven
  * يسمح بإضافة مرحلة جديدة دون تعديل منطق اللعبة الأساسي

## ========================

## 4️⃣ الالتزام الصارم ببنية المجلدات (إجباري)

## ========================

❗ يُمنع منعًا باتًا إنشاء ملفات خارج هذه البنية ❗
❗ أي خرق = خطأ قاتل ❗

```
app
├── src
│   ├── main
│   │   ├── java/com/game/اسم_اللعبة/devimpact
│   │   │   ├── core
│   │   │   ├── engine
│   │   │   ├── rendering
│   │   │   ├── gameplay
│   │   │   │   ├── player
│   │   │   │   ├── enemies
│   │   │   │   ├── mechanics
│   │   │   │   └── levels
│   │   │   ├── managers
│   │   │   ├── ui
│   │   │   ├── data
│   │   │   └── utils
│   │   ├── res
│   │   │   ├── layout
│   │   │   ├── values
│   │   │   ├── drawable
│   │   │   └── raw
│   │   └── AndroidManifest.xml
```

* كل Class يجب أن يكون في المجلد الصحيح.
* يجب **شرح سبب وجود كل ملف في مكانه**.
* لا Logic في UI Layer.
* لا Rendering في Domain Layer.
