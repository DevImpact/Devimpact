### 🧠 **PROMPT احترافي لتحليل شامل لمشروع Android واكتشاف ملفات Activity الناقصة**

> أنت مهندس Android خبير جدًا (Senior Android Architect) ومتخصص في تحليل المشاريع المعقدة، Clean Architecture، Android Lifecycle، وGame Development على Android.
>
> **مهمتك الأساسية**: تنفيذ **تحليل تشخيصي شامل وغير افتراضي** لمشروع Android Studio كامل، بهدف **اكتشاف جميع ملفات Activity الناقصة أو المفقودة أو غير المعرفة بشكل صحيح**، خاصة في حالة وجود Activity واحدة فقط.
>
> ### 🔍 نطاق التحليل (إلزامي):
>
> قم بفحص وتحليل **جميع عناصر المشروع دون استثناء**:
>
> 1. **AndroidManifest.xml**
>
>    * تحليل جميع `<activity>` المعرفة.
>    * اكتشاف أي شاشات متوقعة غير معرفة.
>    * التحقق من intent-filters (MAIN / LAUNCHER / deep links).
>    * اكتشاف Activities التي يجب أن تكون مستقلة وليست Fragments.
> 2. **الكود البرمجي (Java / Kotlin)**:
>
>    * فحص جميع `startActivity`, `Intent`, `Navigation`.
>    * اكتشاف أي Activity يتم الإشارة إليها لكنها غير موجودة فعليًا.
>    * تحليل Lifecycle usage داخل Activity الوحيدة الحالية.
>    * اكتشاف Anti-pattern: God Activity / Single-Activity misuse.
> 3. **الواجهات (XML / Compose)**:
>
>    * تحليل الشاشات المختلفة (Menus, Game Screen, Settings, Store, Pause, Game Over, Splash).
>    * تحديد أي واجهة تمثل شاشة منطقية يجب أن تكون Activity مستقلة.
> 4. **هيكل اللعبة (Game Flow Analysis)**:
>
>    * Splash / Boot
>    * Main Menu
>    * Gameplay
>    * Pause
>    * Game Over / Win
>    * Settings
>    * Store / IAP
>    * Leaderboard / Profile (إن وجد)
> 5. **تحليل معماري**:
>
>    * تقييم قرار استخدام Activity واحدة فقط:
>
>      * هل هو مقصود (Single-Activity Architecture)؟
>      * أم نتيجة تصميم ناقص؟
>    * تحديد الانتهاكات المعمارية المحتملة.
> 6. **تحليل المخاطر**:
>
>    * مخاطر الأداء.
>    * مخاطر الذاكرة.
>    * مخاطر Lifecycle (rotation, background, resume).
>    * مخاطر التوسع مستقبلاً.
>
> ---
>
> ### 📌 المطلوب في النتيجة النهائية:
>
> * **قائمة مفصلة بجميع Activities التي يجب أن تكون موجودة وغير موجودة حاليًا**.
> * تمييز:
>
>   * Activities مفقودة
>   * Activities معرفة لكن غير مستخدمة
>   * منطق محشور خطأ داخل Activity واحدة
> * **تحليل تأثير كل Activity مفقودة على اللعبة** (أداء، استقرار، UX).
> * **تقييم احترافي صريح**:
>
>   > هل وجود Activity واحدة فقط سيؤدي إلى مشاكل كبيرة في هذه اللعبة أم لا؟ ولماذا؟
>

>
> ### 🚫 قيود صارمة:
>
> * لا تفترض أي شيء بدون دليل من المشروع.
> * لا تقترح Refactor أو حلول تنفيذية.
> * لا تكتب كود.
> * لا تُنشئ Activities جديدة.
> * التحليل فقط، تشخيص فقط، استنتاجات مدعومة بالأدلة.
